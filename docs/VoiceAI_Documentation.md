## 1. Обзор продукта

VoiceScribe — локальное приложение для Windows, которое преобразует голос в текст без отправки данных в интернет. Работает полностью на GPU пользователя, обеспечивая конфиденциальность и скорость.

### 1.1 Ключевая идея

Нажал горячую клавишу — говоришь — отпустил — получил чистый готовый текст в буфере обмена. Никакого облака, никаких подписок, никакой задержки сети.

> **Целевой сценарий:** Диктовка заметок, сообщений, документов в любом приложении Windows. Нажал `Ctrl+Shift+Space`, продиктовал, вставил через `Ctrl+V`.

### 1.2 Ключевые характеристики

- Полностью офлайн — данные не покидают компьютер
- Сквозная задержка менее 1 секунды на RTX 5070 Ti
- Поддержка русского и английского языков (автоопределение)
- Умная постобработка: убирает слова-паразиты, расставляет пунктуацию
- Два режима записи: удержание клавиши и переключение (свободные руки)
- Системный трей — приложение не мешает основной работе

## 2. Технический стек

### 2.1 Компоненты

| Компонент | Технология | VRAM | Роль |
|-----------|-----------|------|------|
| Запись звука | sounddevice + numpy | CPU | Захват PCM 16kHz mono |
| VAD | Silero VAD v4 | CPU | Детекция речи / тишины |
| STT (основной) | Faster-Whisper large-v3-turbo INT8 | ~1.5 ГБ | Распознавание речи |
| LLM (редактор) | Phi-3.5 Mini Q4_K_M (llama-cpp) | ~2.2 ГБ | Постобработка текста |
| Горячие клавиши | pynput | — | Глобальный перехват |
| UI / Трей | PyQt6 | — | Системный трей + меню |
| Буфер обмена | pyperclip | — | Копирование текста |
| Уведомления | plyer (Windows Toast) | — | Toast-уведомления |

> **Итого VRAM:** ~3.7 ГБ из 12 ГБ (RTX 5070 Ti). Свободный запас ~8 ГБ.

### 2.2 Требования к системе

| Параметр | Значение | Описание |
|----------|----------|----------|
| ОС | Windows 11 | Windows 10 (21H2+) также поддерживается |
| GPU | NVIDIA RTX (Blackwell+) | Минимум 6 ГБ VRAM, рекомендуется 8+ ГБ |
| Драйвер | >= 572.x | Обязательно для архитектуры Blackwell |
| CUDA | >= 12.8 | Совместимость с CTranslate2 >= 4.7.1 |
| CTranslate2 | >= 4.7.1 | Стабильная поддержка Blackwell с февраля 2026 |
| Python | 3.11 или 3.12 | 3.13 пока не поддерживается llama-cpp |
| RAM | >= 16 ГБ | 8 ГБ минимум, 16 ГБ рекомендуется |
| Микрофон | Любой | Встроенный, USB, Bluetooth |

## 3. Пайплайн обработки

Полный путь от нажатия клавиши до текста в буфере обмена:

| № | Этап | Описание |
|---|------|----------|
| 1 | Горячая клавиша | Пользователь нажимает `Ctrl+Shift+Space`. Иконка трея меняется на красную (запись). |
| 2 | Захват аудио | sounddevice начинает запись PCM 16kHz mono в кольцевой буфер. |
| 3 | Silero VAD | Детектор речи работает в реальном времени, отсекает тишину и фоновый шум. |
| 4 | Окончание записи | Пользователь отпускает клавишу (hold) или нажимает повторно (toggle). Иконка — жёлтая (обработка). |
| 5 | Faster-Whisper STT | Аудио-сегмент транскрибируется на GPU. Язык определяется автоматически (RU/EN). |
| 6 | Phi-3.5 Mini | LLM убирает слова-паразиты, расставляет пунктуацию, структурирует текст. |
| 7 | Буфер обмена | Готовый текст копируется через pyperclip. Иконка — зелёная (готово). |
| 8 | Toast-уведомление | Windows Toast: "Скопировано!" + первые 60 символов текста. Иконка возвращается в синюю. |

## 4. Иконка в системном трее

### 4.1 Состояния иконки

Иконка всегда отражает текущее состояние приложения. Цвет + форма — без текста, чтобы работало на любом размере трея.

| Иконка | Состояние | Тултип / поведение |
|--------|-----------|-------------------|
| 🔵 | Ожидание | VoiceScribe готов. Кликни правой кнопкой для меню. Подсказка: текущий режим горячей клавиши. |
| 🔴 | Запись | Идёт запись. Пульсирующая анимация. Тултип: длительность записи (0:03, 0:04...). |
| 🟡 | Обработка | Whisper + Phi обрабатывают аудио. Тултип: "Распознаю..." или "Редактирую...". |
| 🟢 | Готово | Текст скопирован в буфер. Держится 2 секунды, затем возвращается синяя. Тултип: первые 40 символов. |
| ⚪ | Пауза / Mute | Приложение на паузе (из меню). Горячая клавиша временно отключена. |
| 🔴❗ | Ошибка | Что-то пошло не так. Кликнуть для деталей. Тултип: краткое описание ошибки. |

### 4.2 Контекстное меню (правая кнопка)

Меню трея разделено на логические секции:

| Пункт меню | Действие |
|-----------|----------|
| **Режим: Удержание (Hold)** | Переключить на режим Toggle. Галочка у активного. |
| **Режим: Свободные руки (Toggle)** | Переключить на режим Hold. Галочка у активного. |
| — | *Разделитель* |
| **Горячая клавиша...** | Открыть мини-окно переназначения клавиши. |
| **Настройки...** | Открыть окно настроек. |
| — | *Разделитель* |
| **Пауза / Возобновить** | Временно отключить горячую клавишу. |
| **Выход** | Завершить приложение. |

## 5. Режимы записи и горячая клавиша

### 5.1 Режим Hold — Удержание

> **Принцип:** Нажал и держишь — запись идёт. Отпустил — запись останавливается, текст обрабатывается.

- Лучший выбор для коротких фраз и быстрых заметок
- Нет риска случайно оставить запись включённой
- Ощущение похоже на рацию — интуитивно понятно
- Максимальная длина записи: 5 минут (настраивается)
- Если отпустил раньше чем через 0.5с — запись игнорируется (защита от случайного нажатия)

### 5.2 Режим Toggle — Свободные руки

> **Принцип:** Первое нажатие — начало записи. Второе нажатие — конец записи и обработка.

- Подходит для длинных диктовок, когда руки нужны для других задач
- Иконка пульсирует красным всё время записи — напоминание что микрофон открыт
- Автостоп: если речь не обнаружена VAD более 30 секунд — запись останавливается автоматически
- Toast-предупреждение при автостопе: "Запись остановлена по тишине"

### 5.3 Назначение горячей клавиши

Мини-окно переназначения открывается из меню трея или через Настройки.

| Параметр | Детали |
|----------|--------|
| По умолчанию | `Ctrl + Shift + Space` |
| Допустимые модификаторы | Ctrl, Shift, Alt, Win (один или несколько) |
| Основная клавиша | Любая буква, цифра, F1–F12, Space, Tab |
| Запрещённые комбинации | `Ctrl+C` / `Ctrl+V` / `Ctrl+Z` (системные) |
| Применение | Мгновенно, без перезапуска |
| Сохранение | В `config.yaml`, сохраняется между сессиями |

## 6. Окно настроек

Настройки хранятся в `config.yaml` и доступны через окно настроек (меню трея → Настройки...).

### 6.1 Секция: Горячая клавиша и режим

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `hotkey.key` | `ctrl+shift+space` | Комбинация клавиш для запуска записи |
| `hotkey.mode` | `hold` | `hold` — удержание, `toggle` — переключение |
| `hotkey.min_hold_ms` | `500` | Минимальное время нажатия (мс), Hold режим |
| `hotkey.autostop_sec` | `30` | Автостоп по тишине в Toggle режиме (сек) |
| `hotkey.max_record_sec` | `300` | Максимальная длина записи (5 минут) |

### 6.2 Секция: Аудио

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `audio.device` | `default` | Устройство ввода (`default` = системное) |
| `audio.sample_rate` | `16000` | Частота дискретизации, Гц. Не менять. |
| `audio.channels` | `1` | Моно. Не менять. |
| `audio.chunk_ms` | `30` | Размер блока VAD в миллисекундах |

### 6.3 Секция: Модели

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `transcriber.model` | `large-v3-turbo` | Размер модели Whisper |
| `transcriber.device` | `cuda` | `cuda` или `cpu` |
| `transcriber.compute_type` | `int8` | `int8` (быстро) или `float16` (надёжнее) |
| `editor.model_path` | `models/Phi-3.5-mini-Q4.gguf` | Путь к GGUF файлу Phi-3.5 Mini |
| `editor.gpu_layers` | `-1` | `-1` = все слои на GPU |
| `editor.enabled` | `true` | `false` = только сырая транскрипция, без LLM |

### 6.4 Секция: Вывод

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `output.mode` | `clipboard` | `clipboard` = только буфер обмена |
| `output.notify` | `true` | Показывать Toast-уведомление |
| `output.notify_duration_ms` | `3000` | Длительность уведомления (мс) |
| `output.history_size` | `0` | История отключена — текст только в буфер обмена |

## 7. Экран приветствия (Onboarding)

При первом запуске приложения отображается полноэкранный экран приветствия. При повторных запусках — пропускается, приложение уходит сразу в трей.

### 7.1 Структура экрана

| Зона | Содержание |
|------|-----------|
| **Логотип** | Анимированный логотип VoiceScribe — звуковая волна трансформируется в текст. Синий + фиолетовый градиент. |
| **Заголовок** | «VoiceScribe» крупно. Подзаголовок: «Голос → Текст. Локально. Мгновенно.» |
| **Загрузка** | Прогресс-бар загрузки моделей с текстом этапа: «Загрузка Whisper...» → «Загрузка Phi-3.5...» → «Готово!» |
| **Горячая клавиша** | Крупная визуальная подсказка: иконки клавиш `Ctrl + Shift + Space` с анимацией нажатия. |
| **Режим** | Две карточки с иконками: Hold (рука с кнопкой) и Toggle (переключатель). Активный режим подсвечен. |
| **Кнопка** | «Начать работу» — закрывает экран, приложение уходит в трей. Больше не показывается. |

### 7.2 Визуальный стиль

- Тёмный фон (`#0F172A`) с тонкой сеткой точек — глубина и технологичность
- Логотип: звуковая волна (синий `#2563EB`) плавно трансформируется в буквы «VS»
- Анимация входа: элементы появляются снизу вверх с fade-in, 600мс
- Прогресс-бар: градиент синий → фиолетовый, заполняется по мере загрузки моделей
- Карточки режимов: стекло-морфизм (rgba белый с blur), рамка при выборе активного
- Кнопка «Начать»: градиент синий→фиолетовый, hover-эффект с glow

> **Технически:** Экран реализован на PyQt6 (`QDialog`, fullscreen). Факт первого запуска сохраняется в `config.yaml` (`first_run: false`). Загрузка моделей происходит в фоновом потоке с передачей прогресса в UI через Qt signals.

## 8. Постобработка через Phi-3.5 Mini

### 8.1 Что делает LLM

Phi-3.5 Mini получает сырую транскрипцию от Whisper и применяет следующие правила:

- Удаляет слова-паразиты: «ну», «вот», «как бы», «значит», «э-э», «ммм», «короче»
- Расставляет запятые, точки, знаки вопроса и восклицания
- Делает заглавными первые буквы предложений и имён собственных
- Разбивает на абзацы по смыслу если текст длиннее 3 предложений
- Исправляет явные ошибки распознавания (контекстно)
- **НЕ меняет смысл, НЕ добавляет слова от себя**
- Язык вывода совпадает с языком ввода

### 8.2 Пример работы

| До (сырая транскрипция) | После (Phi-3.5 Mini) |
|------------------------|---------------------|
| *ну вот значит нужно э-э сделать отчёт по проекту ну там три раздела значит введение основная часть ну и заключение* | Нужно сделать отчёт по проекту. Три раздела: введение, основная часть и заключение. |

> **Опция:** Постобработку можно отключить в настройках (`editor.enabled = false`). В этом режиме текст копируется сразу после Whisper без LLM.

## 10. Установка и первый запуск

### 10.1 Предварительные требования

- Установить драйвер NVIDIA >= 572.x
- Установить CUDA Toolkit 12.8: [developer.nvidia.com/cuda-downloads](https://developer.nvidia.com/cuda-downloads)
- Установить Python 3.11 или 3.12: [python.org](https://python.org)
- Установить Git: [git-scm.com](https://git-scm.com)

### 10.2 Установка

**Шаг 1 — клонировать репозиторий:**

```bash
git clone https://github.com/your-org/voicescribe.git
cd voicescribe
```

**Шаг 2 — запустить автоустановку:**

```bash
python setup.py
```

`setup.py` автоматически:

- Проверит версию драйвера, CUDA и CTranslate2
- Создаст виртуальное окружение (`.venv`)
- Установит все зависимости включая `llama-cpp-python` с флагом CUDA
- Скачает модели Faster-Whisper и Phi-3.5 Mini (~4 ГБ, только первый раз)

**Шаг 3 — запустить:**

```bash
python main.py
```

> **Первый запуск:** При первом запуске модели загружаются в VRAM (~15–30 секунд). Последующие запуски быстрее — модели кэшируются на диске.

## 12. Возможные улучшения (после MVP)

Функции не входящие в MVP, но потенциально полезные:

- Автовставка текста в активное поле ввода (как Aqua Voice) — через `pyautogui`
- Трёхуровневый fallback: INT8 → float16 → whisper.cpp CPU
- Индикатор уровня громкости на экране при записи (звуковая волна)
- Профили постобработки: «Заметки», «Email», «Код» — разные промпты
- Транскрипция аудиофайлов через перетаскивание на иконку трея
- История последних N записей в меню трея
- Автозапуск при входе в Windows (опция в настройках)
